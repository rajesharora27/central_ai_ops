# Global Artifact and Document Governance

## Mandatory Storage Rules

All AI-generated plans, tasks, architecture notes, implementation strategies, and task ledgers must live inside the governed project repository under version control.

### Where things go

| Artifact | Location | Notes |
|----------|----------|-------|
| Plans | `docs/plans/` | Required before repo-changing implementation work |
| Tasks | `docs/tasks/` | Full implementation task set |
| Completed tasks | `docs/tasks/completed/` | Archive completed task files; do not delete them |
| Task hub | `docs/TODO.md` | Auto-generated by `npm run tasks:sync` |
| Product / engineering docs | `docs/*.md` | Canonical behavior, architecture, workflow, security, and release docs |

### Where things never go

- Never store project plans or tasks only in tool-specific folders such as `.claude/`, `.cursor/`, `.gemini/`, `.antigravity/`, `~/.claude`, or temp directories.
- Never leave implementation plans only in chat context.
- Never treat `central_ai_ops` as the storage location for project-specific plans or tasks; it is the governance source, not the project ledger.

## Mandatory Preflight

Before any repo-tracked implementation change:

1. Reuse an existing plan in `docs/plans/` if it already covers the request; otherwise create or update one.
2. Reuse or create the full implementation task set in `docs/tasks/`.
3. Run `npm run tasks:sync` so `docs/TODO.md` reflects the current task tree.
4. Only then begin implementation.

This rule applies to features, fixes, docs changes with implementation impact, refactors, migrations, release automation, and patch-by-patch delivery. It does not apply to pure Q&A or analysis with no repo mutation.
